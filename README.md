<h1>Задача</h1>
<p>Даны прямоугольники на плоскости с углами в целочисленных координатах ([1..109],[1..109]).
Требуется как можно быстрее выдавать ответ на вопрос «Скольким прямоугольникам принадлежит точка (x,y)?» И подготовка данных должна занимать мало времени.
Только нижние границы включены => (x1<= x) && (x<x2) && (y1<=y) && (y<y2)</p>

<h1>Цель</h1>
<p>Реализовать три разных решения задачи
Выяснить при каком объеме начальных данных и точек какой алгоритм эффективнее.
</p>

<h1><a href='algorithms_lab2/algorithms.py'>Реализация алгоритмов</a></h1>
<h3>Перебор</h3>
<p>Для начала я написала алгоритм простого перебора, где мы просто проходимся по каждому прямоугольнику и смотрим, принадлежит ли ему данная точка. Асимптотика этого алгоритма: подготовка - O(1), поиск точек - O(n)</p>
<h3>Карта</h3>
<p>Далее я написала алгоритм, который сжимает координаты и строит матрицу, которая хранит количество прямоугольников. Далее смотрим какому промежутку принадлежит каждая точчка и находим ее в матрице. Асимптотика этого алгоритма: подготовка - O(n^3), поиск точек - O(logn)</p>
<h3>Персистентное дерево отрезков</h3>
<p>Чтобы реализовать этот алгоритм понадобиться написать <a href='algorithms_lab2/persistentTree.py'>реализацию пресистентного дерева</a>, которая содержит в себе класс Node, она хранит ссылки на детей и значение ноды, и класс PersistentTree,
который состоит из функций:  __init__, build, update, get, get_ans, add_version. Подготовка данных в этом алгоритме происходит за счет сжатия координат по y и построению по ним персистентного дерева отрезков.
Для поиска точки мы находим нужную версию дерева с помощью бинарного поиска и пользуемся функцией get_ans нашего класса PersistentTree для получения ответа. Подготовка занимает O(nlogn), а поиск O(logn).
</p>
